## 🎯 战略调整：Self-Bootstrap 优先

### 为什么要优先实现 Self-Bootstrap？

| 原因 | 说明 |
|------|------|
| **核心竞争力** | Self-Bootstrap 是 IntentOS 区别于其他 AI 系统的核心特征 |
| **自举能力** | 系统可以用意图管理自身，实现真正的 AI-Native |
| **自演化** | 系统可以从使用中学习并改进 |
| **差异化** | 这是 IntentOS 的独特价值主张 |

### 原路线图的问题

```
原计划:
v0.6.0: 意图图谱 + 形式化验证 (功能增强)
v0.7.0: 意图学习 (自演化)
v0.8.0: Self-Bootstrap (核心) ← 太晚了！

调整后:
v0.6.0: Self-Bootstrap (核心) ← 提前到第一个里程碑
v0.7.0: 意图图谱 + 形式化验证 (功能增强)
v0.8.0: 意图学习 + 性能优化 (自优化)
```

## 短期目标 (v0.6.0 - 1 个月) 🔴 Self-Bootstrap 优先

### 1. Self-Bootstrap 核心实现 🔴 最高优先级

**目标**: 系统可以用意图管理自身

**任务**:

#### 1.1 元意图 DSL 定义
- [ ] 定义元意图语法
  ```yaml
  meta_intent:
    action: create_template | modify_policy | register_capability
    target: intent_template | policy | capability
    parameters: {...}
  ```
- [ ] 定义元意图执行语义
- [ ] 编写元意图规范文档

**预期代码**: ~500 行

#### 1.2 系统自省 API
- [ ] 实现意图模板查询
  ```python
  templates = await registry.query_templates(tags=["sales"])
  ```
- [ ] 实现能力注册查询
  ```python
  capabilities = await registry.list_capabilities()
  ```
- [ ] 实现执行历史查询
  ```python
  history = await engine.get_execution_history(intent_id="...")
  ```
- [ ] 实现系统状态查询
  ```python
  status = await system.get_status()
  ```

**预期代码**: ~800 行

#### 1.3 系统自修改
- [ ] 实现创建新意图模板
  ```python
  await registry.create_template(
      name="new_template",
      template=template_def,
  )
  ```
- [ ] 实现修改执行策略
  ```python
  await engine.update_policy(
      intent_type="sales_analysis",
      policy={"timeout": 600},
  )
  ```
- [ ] 实现注册新能力
  ```python
  await registry.register_capability(
      name="new_capability",
      capability=cap_def,
  )
  ```
- [ ] 实现回滚机制
  ```python
  await system.rollback(change_id="...")
  ```

**预期代码**: ~1,000 行

#### 1.4 安全边界
- [ ] 实现权限分级
  ```python
  @require_permission("admin")
  async def create_template(...):
      ...
  ```
- [ ] 实现操作审计日志
  ```python
  await audit.log(
      action="create_template",
      user=context.user_id,
      details=...,
  )
  ```
- [ ] 实现敏感操作多人审批
  ```python
  if action in ["delete_template", "modify_policy"]:
      await request_approval(approvers=["admin1", "admin2"])
  ```

**预期代码**: ~700 行

**预期代码总计**: ~3,000 行  
**测试用例**: 50+  
**预计完成**: 2026-04-12
