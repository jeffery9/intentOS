# IntentOS 分层架构 (v2.1 融合版)

> **语义内核为圆心 · IO 能力为半径 · PaaS 生态为周界**

---

## 一、 整体架构

IntentOS 采用严格的**语义解耦架构**，将“智力逻辑”与“物理执行”分离。

### 1.1 核心分层拓扑

```
┌─────────────────────────────────────────────────────────────────┐
│              PaaS 服务层 (应用市场 / 计费)                      │
│    • 对话式 App 生成 • 自动 Gas 结算 • 租户资源管理 • 市场交易      │
└────────────────────┬────────────────────────────────────────────┘
                     │ OS API (v1 REST / WebSocket)
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│              IO 能力层 (AI Agent / 物理驱动)                    │
│    • Capability Gate • MCP 集成 • Shell • 物理资源映射            │
└────────────────────┬────────────────────────────────────────────┘
                     │ 语义指令驱动 (Opcode -> Handler)
                     ▼
┌─────────────────────────────────────────────────────────────────┐
│              语义内核层 (The Kernel / SVM)                      │
│    • Semantic VM Core • Gas System • Watchdog • Distributed Mem │
└─────────────────────────────────────────────────────────────────┘
```

---

## 二、 语义内核层

内核是 IntentOS 的“主板”，负责处理语义指令流。

- **语义 VM (SVM)**: 执行语义操作码。
- **LLM 处理器**: 作为语义 CPU，负责自然语言的推理与指令拆解。
- **Gas 系统**: 实现物理消耗的统一计量。
- **Watchdog**: 实时健康哨兵，确保处理器逻辑的完整性。
- **分布式内存**: 跨节点的、带命名空间隔离的全局语义存储。

---

## 三、 IO 能力层

Agent 扮演“驱动程序”角色，是内核触达物理世界的唯一通道。

- **能力注册中心**: 注册所有可用的物理工具。
- **能力门控**: 基于上下文权限的准入控制。
- **私有资源嫁接**: 将租户的私有物理资源安全地映射为语义能力。
- **隔离执行**: 确保不同租户的 IO 操作在物理和语义层面均被隔离。

---

## 四、 PaaS 服务层

PaaS 是 IntentOS 的商业化外壳，负责生态治理。

- **应用市场**: AI Native App 的发布、搜索与分发中心。
- **App 生成器**: 结合 `AppConversationStudio`，实现“对话即开发”的 App 生长模式。
- **计费与收益**: 将内核产生的 Gas 消耗自动转化为商业账单，并向开发者分成。
- **租户管理器**: 管理租户的混合配额。

---

## 六、 架构演进原则

1. **计算与执行分离**: 内核只负责语义计算，不直接处理物理 IO。
2. **逻辑与状态分离**: App 定义是静态的，App 实例是动态绑定的。
3. **收益驱动进化**: 每一单位 Gas 消耗都产生收益，收益用于物理节点的自动扩容。
