Welcome to IntentOS!

IntentOS is a revolutionary AI-native operating system designed to be autonomous, self-reproducing, and self-sustaining. This guide will walk you through the initial steps to deploy your IntentOS instance in the cloud.

Once launched, IntentOS will manage its own cloud resources, generate revenue, and even reproduce itself, all while adhering to its core mission and ethical guidelines. Let's get started!

Before You Begin

Ensure you have the following prerequisites installed and configured on your system:

  • Python 3.9+: Download Python
  • pip (Python package installer): Usually comes with Python.
  • Cloud Provider CLI (e.g., AWS CLI):

    For AWS, install the AWS CLI: AWS CLI Installation Guide

    Other providers (GCP, Azure) will have similar CLI tools.

  • Docker / Docker Compose (Optional, for local testing): Get Docker

Step 1: Configure Cloud Credentials

IntentOS needs access to your cloud account to manage resources. We highly recommend configuring credentials securely. IntentOS supports interactive guidance for this step.

Option A: Using AWS CLI (Recommended)

If you have the AWS CLI installed, open your terminal and run:

aws configure

Follow the prompts to enter your AWS Access Key ID, AWS Secret Access Key, default region (e.g., us-east-1), and preferred output format (e.g., json).

Option B: Using Environment Variables

Alternatively, you can set credentials as environment variables:

export AWS_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID"
export AWS_SECRET_ACCESS_KEY="YOUR_SECRET_ACCESS_KEY"
export AWS_DEFAULT_REGION="us-east-1"

Note: Replace "YOUR_ACCESS_KEY_ID" and "YOUR_SECRET_ACCESS_KEY" with your actual credentials.

Step 2: Run IntentOS Cloud Orchestrator

Navigate to the root directory of your IntentOS project in your terminal.

The CloudOrchestrator will automatically detect your configured cloud provider, load resource definitions (from cloud/[provider]/resources.yml), generate a deployment plan, and estimate costs. It will then ask for your confirmation before making any changes.

python -m intentos.bootstrap.cloud_orchestrator

Important: Review the deployment plan and estimated costs carefully. IntentOS prioritizes sustainability and will always inform you of potential expenses. Confirm with 'yes' to proceed.

If credentials are not detected or are invalid, the orchestrator will provide interactive guidance, similar to the instructions above, and wait for you to configure them before proceeding.

Step 3: Verify Your Deployment

After the orchestrator successfully executes the plan, IntentOS will be running in your cloud environment. You can verify its status by:

  • Checking your cloud provider's console (e.g., AWS ECS, EC2, Redis dashboards).
  • Accessing the IntentOS endpoint (if exposed). The orchestrator output will provide details.
  • (Future) Running a simple status check command for your IntentOS instance.

Congratulations! Your IntentOS digital organism is now alive and operating autonomously.
