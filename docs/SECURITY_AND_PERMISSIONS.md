# IntentOS 安全与权限架构 (v2.1 融合版)

> **语义隔离 · 能力门控 · 双模态执行 · 沙箱防护**

---

## 一、 核心安全模型

IntentOS 采用**语义级安全隔离**，防止通过自然语言指令绕过传统安全层。

### 1.1 语义命名空间
所有存储在分布式内存（Semantic Memory）中的对象均强制隔离：
- **格式**: `tenant_id:user_id:key`
- **隔离级别**: 任何用户进程只能访问其所属命名空间的数据。

### 1.2 双模态执行
- **用户态**: 默认执行模式。限制访问系统配置，受 Gas 配额约束。
- **内核态**: 仅供系统组件使用，允许执行特权指令。

---

## 二、 能力门控

在 IO 层（AI Agent），所有物理操作必须通过 **Capability Gate**。

### 2.1 权限声明与校验
App 在 `manifest.yaml` 中声明所需权限，`CapabilityRegistry` 在运行时校验 `AgentContext` 中的权限令牌。

### 2.2 核心权限分类
| 权限 ID | 描述 | 风险等级 |
| :--- | :--- | :--- |
| `system:shell` | 执行物理 Shell 命令 | 极高 |
| `io:fs:read` | 读取本地物理文件 | 高 |
| `io:fs:write` | 写入本地物理文件 | 极高 |
| `net:api:call` | 调用外部网络 API | 中 |
| `kernel:modify` | 修改系统 Prompt 或指令集 | 灾难级 |

---

## 三、 沙箱执行

对于 App 提供的自定义代码或脚本，IntentOS 在沙箱环境中执行：
- **资源限制**: 严格限制 CPU、内存、磁盘和网络。
- **系统调用过滤**: 禁止 `execve`, `fork` 等高危系统调用。
- **文件系统隔离**: 将 App 文件系统根目录 Chroot 到隔离环境。

---

## 四、 审计与自愈

### 4.1 语义审计日志
内核实时记录每一条语义指令的操作人、操作码及结果。

### 4.2 Watchdog 监控
如果检测到 App 试图进行恶意注入或非法提权，Watchdog 会立即中断进程并回滚内核状态。
